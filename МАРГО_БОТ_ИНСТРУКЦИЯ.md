# 🔮 Бот MARGO KARAT — запуск и работа 24/7

Бот выдаёт одноразовые коды `MARGO-FREE-XXXX` для бесплатного вопроса на сайте.
Файл бота: `margo_bot.py` (старый «Славик» в `bot.py` не тронут и работает отдельно).

---

## Шаг 1. Создать бота в Telegram (5 минут)

1. В Telegram откройте **@BotFather** → команда `/newbot`.
2. Имя: `MARGO KARAT`, ник — например `margo_karat_bot` (должен заканчиваться на `bot`).
3. BotFather пришлёт **токен** — скопируйте.

## Шаг 2. Вписать ключи в `.env`

Откройте (или создайте) файл `.env` в этой папке и добавьте строки:

```
MARGO_BOT_TOKEN=токен_от_BotFather
ADMIN_CHAT_ID=ваш_chat_id
```

`ADMIN_CHAT_ID` — чтобы Маргарита получала уведомление о каждом выданном коде.
Узнать свой chat_id: напишите боту **@userinfobot** — он пришлёт число.

## Шаг 3. Проверить локально

Двойной клик по **`start_margo_bot.bat`** → в Telegram напишите боту `/start`.
Пройдите путь: подписки → «Я подписался(-ась)» → ник Instagram → получите код.
Код введите на сайте в разделе «Бесплатный вопрос».

## Шаг 4. Вставить ссылку на бота в сайт

В `index.html` найдите `https://t.me/` (кнопка «Перейти в Telegram» и футер)
и замените на `https://t.me/ваш_ник_бота`.

# ☁️ Работа 24/7 — бесплатное облако

Пока бот запущен через `.bat`, он работает только при включённом компьютере.
Чтобы работал всегда — переносим в облако. Лучший бесплатный вариант:

## Oracle Cloud Free Tier (бесплатно навсегда)

1. Регистрация: https://signup.oraclecloud.com (карта нужна только для проверки, деньги не списывают).
2. Создайте VM: **Compute → Create Instance** → образ **Ubuntu 22.04**, форма **VM.Standard.E2.1.Micro** (Always Free).
3. Скачайте SSH-ключ при создании, подключитесь: `ssh ubuntu@IP_адрес`.
4. На сервере выполните:

```bash
sudo apt update && sudo apt install -y python3-pip python3-venv git
git clone https://github.com/Slavikk99/margo-karat.git   # или загрузите файлы вручную
mkdir -p ~/margo-bot && cd ~/margo-bot
# скопируйте сюда margo_bot.py и requirements.txt (например через scp)
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
nano .env    # вставьте MARGO_BOT_TOKEN и ADMIN_CHAT_ID
```

5. Автозапуск (systemd) — бот сам поднимется после перезагрузки сервера:

```bash
sudo tee /etc/systemd/system/margo-bot.service << 'EOF'
[Unit]
Description=MARGO KARAT Telegram Bot
After=network.target

[Service]
User=ubuntu
WorkingDirectory=/home/ubuntu/margo-bot
ExecStart=/home/ubuntu/margo-bot/venv/bin/python margo_bot.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF
sudo systemctl enable --now margo-bot
sudo systemctl status margo-bot   # должно быть: active (running)
```

Готово — бот живёт в облаке 24/7.

### Альтернативы
- **Docker**: в папке есть `Dockerfile` — `docker build -t margo-bot . && docker run -d --env-file .env --restart always margo-bot` (подойдёт для любого VPS).
- Платные, но простые: Railway.app / Render.com (Background Worker, от ~5 $/мес).

# 🔗 Что дальше (интеграция с сайтом)

Сейчас коды хранятся в `codes.json` рядом с ботом, а сайт проверяет только формат кода.
Когда подключим **Supabase**:
1. Бот будет записывать коды в таблицу `free_codes`.
2. Сайт будет проверять код по-настоящему: существует, не использован, не старше 30 минут.
3. После отправки формы код помечается использованным.

Это уже заложено в коде бота и сайта (см. комментарии TODO).

### Про проверку подписок
Instagram, Facebook и TikTok **не дают** официального API для проверки, подписан ли
конкретный человек. Поэтому используется честная альтернатива: бот просит ник
Instagram и отправляет его Маргарите — можно выборочно проверять. Автопроверка
возможна только для подписки на Telegram-канал (если он появится — добавим).
